# CLAUDE.md — Toms Clippy Light

Instructions for Claude Code (and other AI coding assistants) working on this project.

## Workflow Rules

1. **Always test locally before committing.** Run `make install`, relaunch the app, and verify the affected behaviour manually.
2. **Increment `CFBundleVersion` in `project.yml` with every installed build.** The version string (`CFBundleShortVersionString`) only changes for user-visible releases.
3. **Never edit `TomsClippyLight.xcodeproj` directly.** It is generated by XcodeGen from `project.yml`. After changing `project.yml`, run `xcodegen generate` (or `make generate`).
4. Commit only after local verification. Push and create a release only when explicitly asked.

## Quick Reference

```bash
make install    # build Release, sign, copy to /Applications (the main local workflow)
make test       # run unit + UI tests
make generate   # regenerate .xcodeproj from project.yml
make clean      # wipe build/ and generated project
```

## Project Overview

Native macOS 26 Tahoe menubar clipboard-history app. No Dock icon (`LSUIElement`). Written in Swift 6 (`SWIFT_STRICT_CONCURRENCY: complete`) with SwiftUI + AppKit. One SPM dependency: `KeyboardShortcuts`.

Comprehensive architecture and technical documentation is in [`.github/copilot-instructions.md`](.github/copilot-instructions.md).

## Critical Known Gotchas

These are non-obvious constraints that have caused production bugs. Do not change them without understanding why they exist.

### `KeyablePanel` — must not be replaced with `NSPanel`

`NSPanel` with `.borderless` styleMask returns `false` from `canBecomeKey` by default, which prevents text fields from receiving keyboard input. We use a subclass at the bottom of `AppDelegate.swift`:

```swift
private final class KeyablePanel: NSPanel {
    override var canBecomeKey: Bool { true }
    override var canBecomeMain: Bool { false }
}
```

### `activate()` not `activate(from:)`

This is an `LSUIElement` app. After `historyPanel?.orderOut(nil)`, the app instantly loses frontmost status. `activate(from:)` requires the caller to be frontmost and silently fails at that point. `activate()` (deprecated but functional) has no such requirement and is used intentionally in `WorkspaceProtocol.swift`.

### `captureFocus()` before `NSApp.activate()`

`paster.captureFocus()` must be the first call in `showHistoryPanel()`, before `NSApp.activate(ignoringOtherApps: true)`. Once our panel is active, the previously focused app is no longer frontmost.

### Arrow keys via local `NSEvent` monitor

Up/down arrow navigation is handled by a local `NSEvent.addLocalMonitorForEvents` monitor in `AppDelegate.showHistoryPanel()`, not by SwiftUI focus state. It posts `NotificationCenter` messages (`historyMoveUp`, `historyMoveDown`, `historySelectCurrent`) that `HistoryPopover` receives via `onReceive`. The monitor is torn down in `closeHistoryPanel()`.

### TCC and Apple Development signing

Ad-hoc signing changes the binary hash on every build, which orphans TCC Accessibility grants. The Makefile auto-detects an Apple Development certificate (Team `UVRF68766E`) and signs with it locally. If the cert is absent, auto-paste will not work until Accessibility permission is re-granted.

## Testing

- **Swift Testing** for unit tests (`TomsClippyLightTests/`). Protocol fakes for all system APIs live in `Fakes/`.
- **XCUITest** for Settings window smoke tests (`TomsClippyLightUITests/`).
- Core services are tested with injected fakes — never use `NSPasteboard.general` or `NSWorkspace.shared` directly in tests.

## Localisation

All UI strings go through the `L10n` enum (`TomsClippyLight/Localization/L10n.swift`). The source of truth is `Localizable.xcstrings` (EN base + DE translation). Never use raw string literals for user-visible text.
